## 2. Root-cause analysis

### CR2 = 0%

`pass3a_cards.py:473-505` CR2 prompt requires:
- 3 events from DIFFERENT chunks ≥5 apart
- Each event uniquely describable
- "If you cannot find 3 clearly distinct, well-separated events, output `[]`"

`pass3a_cards.py:1500-1506` CR2 verifier requires:
- C1: 3-event permutation A→B→C
- C2: gold permutation matches CHRONOLOGICAL order in support_chunks
- C3: events UNAMBIGUOUSLY identifiable

**Hypothesis**: teacher generates few CR2 cards (most videos lack clearly distinct, well-separated events), and the strict C2 chronology check rejects most of those.

### CR1 = 12%

`pass3a_cards.py:1491-1498` CR1 verifier C4: "3 distractor options must be SAME-DOMAIN actions visible elsewhere in this video (not invented out of common sense)".

**Hypothesis**: teacher LLM uses common-sense distractors (e.g., for "why is the rider on the motorcycle", distractors include things not in the video). C4 rejects → 88% loss.

### F5 = 37%

F5 needs 3+ contiguous chunks of same action — only ~37% of batch1 videos qualify. Structural data limit, not prompt issue.

### F6 = 50%

F6 needs FUTURE visibility (action that will happen). 50% means half the videos don't have enough setup-then-payoff structure. Structural.

## 3. Proposed batch2 adjustments

### 3.1 Verifier loosening (high impact, safe)

| Family | Current | Proposed | Expected lift |
|--------|---------|----------|--------------|
| **CR1** | C4 requires same-domain distractors | Drop C4; only check causal validity (C1-C3) | 12% → 40-50% |
| **CR2** | C2 requires strict chronology match | Allow chronology check but accept if 2/3 events ordered correctly | 0% → 30-50% |
| **CR3** | Intent inferred from full early-mid video | Keep (already 60%) | — |

### 3.2 Generation prompt clarification (medium impact)

| Family | Change | Rationale |
|--------|--------|-----------|
| CR2 | Allow 2 events instead of 3 (relax to "at least 2 distinct events with order") | More videos qualify |
| CR1 | Add example showing causal pair within video | Teacher better understands C2 same-evidence requirement |
| F5 | Allow 2-chunk repetition fallback | Lower bar for short videos |

### 3.3 Family target rebalancing (counter-cyclical to actual yields)

For batch2, generate **MORE** for low-yield families to compensate:

```python
# v12.1 batch2 multipliers (pass3a_cards.py override)
BATCH2_TARGET_MULTIPLIER = {
    "CR1": 3,   # teacher generates 3, verifier keeps ~1 (compensates 12% yield)
    "CR2": 2,   # teacher generates 2, verifier keeps ~0.5
    "F5":  2,   # teacher generates 2, keeps ~0.7
    "F6":  2,   # teacher generates 2, keeps ~1
    # Other families keep target=1
}
```

LLM cost increase: ~+200% on these 4 families = ~+15% overall pass3a calls.

### 3.4 Batch2 video selection — increase diversity

Batch1 is heavy on YouTube cooking + DIY (high E2/F4 yield, low temporal-reasoning yield).

Batch2 should target:
- **Sports** (clear temporal sequences → CR2 yield ↑)
- **Social/conversation** (causal → CR1 yield ↑)
- **Procedural multi-step** (ASI/SSR coverage)
- **Longer videos** (3-5 min, more chunks → more reasoning context)

Check `select_videos()` filters: currently 60-400s range. Raise upper bound to 600s for batch2 to capture more multi-step procedural content.

### 3.5 Density caps unchanged

v12 caps are already tight (max_traj=5, max_per_traj=3, gap=15). Don't relax for batch2 — density target ~2 q/min is already 2× benchmark median; further loosening harms train→eval distribution match.

## 4. Implementation order

1. **Phase 1 (verifier loosening, free)**: Edit VERIFY_REASONING_PROMPT for CR1 (drop C4) + CR2 (allow 2/3 chronology). No regen needed for batch1; just affects batch2 onwards.
2. **Phase 2 (target multiplier, +15% LLM cost)**: Add `BATCH2_TARGET_MULTIPLIER` knob to pass3a, controlled by env var `THINKSTREAM_BATCH=batch2`.
3. **Phase 3 (video selection diversity)**: Change `select_videos` upper bound 400s → 600s. Add dataset filters for sports/social.
4. **Phase 4 (run batch2)**: Generate 200-400 new videos with adjusted thresholds, append to existing batch1.
5. **Phase 5 (audit)**: Re-run `audit_distribution.py` post-merge; confirm CR1≥30%, CR2≥20%, F5≥50%, F6≥60%, OVO ASI/CRR/REC/FPD coverage ≥60%.

## 5. Threshold sensitivity matrix

| Threshold | v11.5 | v12.0 batch1 | v12.1 batch2 proposed | Risk if too loose |
|-----------|-------|--------------|----------------------|-------------------|
| FAMILY_TARGETS | 25/vid | 26/vid | 26/vid + multipliers | Cost ↑ |
| MAX_TRAJECTORIES | 10 | 5 | 5 | Density drift |
| MAX_QUESTIONS_PER_TRAJ | 6 | 3 | 3 | History overload |
| min_chunk_gap | 8 | 15 | 15 | Density drift |
| video duration upper | 400s | 400s | 600s | Compute cost |
| CR1 verifier C4 | strict | strict | drop | Hallucinated distractors |
| CR2 chronology | strict | strict | 2/3 match | Order errors trained |
| F5 chunk-repetition | 3+ | 3+ | 2+ | Looser REC labels |

## 6. Decision points (user choice required)

1. **Risk vs cost trade**: drop CR1 C4 = +50% yield but admits common-sense distractors. Acceptable?
2. **CR2 2/3 chronology**: trains the model to tolerate one-event-misplaced. Acceptable for ASI?
3. **Batch2 video count**: 200 (low cost) / 400 (medium) / 600 (corpus matches OVO scale)?
4. **Diversity priority**: dataset source mix vs duration extension?

## 7. What to NOT change

- v12 protocol (tools=, multi-turn, inter-chunk compress)
- pass3b density caps (already research-backed)
- pass3c v12 fast path
- pass4 v12 dispatchers
- Split ratios (70/15/15 + 50/50)

These are settled. Batch2 only touches source data quality + family yield.
